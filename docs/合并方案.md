# LOOM × AgentFlow-Studio 合并方案

> 版本：v1.0
> 日期：2026-05-01
> 基础选择：**AgentFlow-Studio**（UI + 后端架构）
> 优化来源：**LOOM**（工作流编辑体验）

---

## 一、合并目标

1. 以 AgentFlow-Studio 为基础骨架
2. 吸收 LOOM 的精华特性
3. 形成一个**生产可用**的 AI 工作流编排平台

### 目标技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端框架 | Vue 3.4+ + Composition API |
| UI 组件库 | Element Plus（保留 AgentFlow 的选择） |
| 状态管理 | Pinia |
| 工作流画布 | Vue Flow |
| 后端框架 | NestJS 11（升级 LOOM 的 10 → 11） |
| 数据库 | PostgreSQL 15+ |
| 缓存 | Redis |
| ORM | TypeORM |
| 认证 | JWT |

## 二、需要合并的模块清单

### 2.1 从 LOOM 吸收到 AgentFlow

| 序号 | 模块 | LOOM 优势 | 合并方式 |
|------|------|----------|----------|
| 1 | 工作流描述 | workflow.description 字段 | 在 workflow entity 添加 description 字段 |
| 2 | Undo/Redo | useWorkflowHistory composable | 移植到前端，集成到 workflow store |
| 3 | Markdown 支持 | knowledge 支持 .md 格式 | 合并 document-parser.ts 的 md 处理逻辑 |
| 4 | 前端样式 | 工业靛蓝 CSS 变量风格 | 将 LOOM 的 CSS 变量体系迁移到 AgentFlow |
| 5 | ChatWidget | 更完善的对话组件 | 吸收 ChatWidget 的布局和交互优化 |
| 6 | 工作流验证 | useWorkflowValidation | 移植到 AgentFlow FlowCanvas |
| 7 | CI 测试 | Jest e2e 测试 | 合并 AgentFlow 的 CI 配置 |

### 2.2 AgentFlow-Studio 保留的核心模块

| 序号 | 模块 | 说明 |
|------|------|----------|
| 1 | PostgreSQL + Redis | 基础设施，不变 |
| 2 | compensation-executor | 错误补偿机制 |
| 3 | code-execution.service | 代码节点执行 |
| 4 | metrics module | 监控统计 |
| 5 | knowledge eval panel | RAG 评估 |
| 6 | JWT 认证 | 用户体系 |

## 三、后端合并详细方案

### 3.1 代码移植清单

#### 3.1.1 实体层（Entity）

```
需要新增/修改的实体：

backend/src/workflow/entities/workflow.entity.ts
  + description: string (从 LOOM 迁移)

backend/src/workflow/entities/workflow-execution.entity.ts
  + status: enum (pending/running/success/failed/compensated)
  + logs: JSON (SSE 流式日志)

backend/src/knowledge/entities/knowledge-base.entity.ts
  + description: string
  + created_at, updated_at (如果缺失)
```

#### 3.1.2 Service 层

```
需要合并的 Service：

1. KnowledgeService 合并
   AgentFlow: knowledge.service.ts (基础CRUD)
   LOOM: chunking.service.ts (更完善的分块)
   LOOM: document-processor.service.ts (支持 Markdown)

   合并策略：以 AgentFlow 为基础，吸收 LOOM 的：
   - Markdown 文档解析
   - 更鲁棒的 text-splitter
   - MIME 类型检测

2. WorkflowService 合并
   AgentFlow: workflow.service.ts (业务逻辑)
   LOOM: executor.service.ts (SSE 流式输出更好)

   合并策略：保留 AgentFlow 的 service 层架构，
   吸收 LOOM executor 的流式日志推送机制

3. ChatService 合并
   AgentFlow: chat.service.ts (基础)
   LOOM: chat.service.ts (人格化对话)

   合并策略：保留 AgentFlow 的 session 管理，
   吸收 LOOM 的对话上下文处理逻辑
```

#### 3.1.3 Module 层

```
backend/src/app.module.ts 合并后结构：

AppModule
├── WorkflowModule (保留 AgentFlow，增加 description 字段)
├── KnowledgeModule (合并 LOOM 的 chunking + 评估)
├── ChatModule (合并 LOOM 的对话逻辑)
├── AgentModule (保留)
├── MetricsModule (保留)
├── HealthModule (保留)
└── AuthModule (JWT，保留但完善)
```

### 3.2 数据库迁移

#### 3.2.1 新增字段

```sql
-- workflow 表新增描述字段
ALTER TABLE workflow ADD COLUMN description TEXT DEFAULT '';
ALTER TABLE workflow ADD COLUMN updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

-- knowledge_base 表新增描述字段
ALTER TABLE knowledge_base ADD COLUMN description TEXT DEFAULT '';

-- workflow_execution 表增强
ALTER TABLE workflow_execution ADD COLUMN logs JSONB DEFAULT '[]';
ALTER TABLE workflow_execution ADD COLUMN compensated_at TIMESTAMP NULL;
```

#### 3.2.2 保留的表结构（来自 AgentFlow）

- `workflow` - 主工作流
- `workflow_execution` - 执行记录
- `document` - 知识库文档
- `document_chunk` - 文档分块
- `knowledge_base` - 知识库
- `chat_session` - 聊天会话
- `chat_message` - 聊天消息
- `metrics_daily` - 每日指标

## 四、前端合并详细方案

### 4.1 代码移植清单

#### 4.1.1 组件移植

| LOOM 组件 | 目标位置 | 合并方式 |
|-----------|----------|----------|
| `useWorkflowHistory.ts` | `frontend/src/composables/workflow/` | 新建，集成到 FlowCanvas |
| `useWorkflowValidation.ts` | `frontend/src/composables/workflow/` | 新建，集成到 NodeConfigDrawer |
| `workflow.types.ts` | `frontend/src/types/` | 合并到 AgentFlow 的 types |
| `style.css` (CSS变量) | `frontend/src/styles/variables.css` | 追加工业靛蓝变量 |
| `ChatWidget.vue` | `frontend/src/components/chat/` | 重构为 ChatMessagesPanel 增强 |

#### 4.1.2 节点类型

```
保留 AgentFlow 的节点：
- TriggerNode (触发节点)
- LLMNode (大模型节点) ✅ LOOM 没有
- CodeNode (代码执行) ✅ LOOM 没有
- ConditionNode (条件分支)
- KnowledgeNode (知识检索)
- HttpNode (HTTP 请求)
- EndNode (结束节点)

从 LOOM 移植：
- AI 回答节点 → 合并到 LLMNode 或新建
- 工作流描述编辑 → 新增 WorkflowDescriptionDrawer
```

#### 4.1.3 Store 合并

```
frontend/src/stores/workflow.ts 增强：
  + history: WorkflowHistory[] (从 useWorkflowHistory 移植)
  + canUndo: boolean
  + canRedo: boolean
  + undo() / redo()

frontend/src/stores/chat.ts：
  合并 LOOM ChatWidget 的对话逻辑
```

### 4.2 UI 布局方案

主布局 (MainLayout.vue) 保留 AgentFlow 结构：
