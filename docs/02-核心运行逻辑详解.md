# 02 - 核心运行逻辑详解

本文档详细讲解 AgentFlow Studio 的三大核心系统的运行逻辑：

1. 工作流系统
2. RAG 知识库系统
3. 对话系统

---

## 一、工作流系统

### 1.1 概念比喻

工作流就像一个**工厂流水线**：

```
原材料进来 → 工位A加工 → 工位B组装 → 质检判断 → 成品输出
     ↓            ↓            ↓           ↓
  触发节点     LLM节点    知识库节点   条件节点
```

### 1.2 支持的节点类型

| 节点图标 | 类型      | 功能         | 比喻         |
| -------- | --------- | ------------ | ------------ |
| ▶️       | trigger   | 接收用户输入 | 工厂大门     |
| 🤖       | llm       | 调用大模型   | AI 工人      |
| 📚       | knowledge | 检索知识库   | 图书馆查资料 |
| 🔀       | condition | 条件判断分支 | 质检分选     |
| 💻       | code      | 执行代码     | 自动化机器   |
| ⏹️       | end       | 结束流程     | 出口         |

### 1.3 工作流执行流程

```
用户点击"执行"
     │
     ▼
┌──────────────────┐
│ 1. 加载工作流定义  │
│    - 读取nodes[]   │
│    - 读取edges[]   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 2. 验证工作流结构  │
│    - 条件节点是否有两条边？
│    - 是否有触发节点？
│    - 是否有环？
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 3. 创建执行记录   │
│    - 状态: pending
│    - 保存input参数
└────────┬─────────┘
         │
         ▼
┌────────────────────────────────────────┐
│ 4. 执行引擎遍历节点                      │
│                                        │
│  trigger ──► llm ──► knowledge ──► end │
│     │        │          │              │
│   接收输入  问AI      查资料          结束 │
│                                        │
└────────┬───────────────────────────────┘
         │
         ▼
┌──────────────────┐
│ 5. 更新执行记录   │
│    - 状态: completed/failed
│    - 保存output结果
│    - 保存执行日志
└──────────────────┘
```

### 1.4 节点执行策略

每个节点可以配置**容错策略**：

```typescript
{
  "retryCount": 3,      // 失败时重试3次
  "retryDelayMs": 1000, // 每次间隔1秒
  "timeoutMs": 30000,   // 最多等待30秒
  "onError": "fail"     // 失败后: fail/skip/rollback/compensate
}
```

**策略说明**：

- **fail**: 立即停止整个工作流
- **skip**: 跳过此节点，继续执行下一个
- **rollback**: 回滚之前已执行的节点
- **compensate**: 执行补偿操作（如删除文件、恢复数据）

### 1.5 条件分支逻辑

```
          ┌─────────────┐
          │  条件节点    │
          │  "分数>60?" │
          └──────┬──────┘
                 │
        ┌────────┴────────┐
     True│              False│
        ▼                 ▼
   ┌─────────┐       ┌─────────┐
   │ 及格处理 │       │ 不及格  │
   └─────────┘       └─────────┘
```

条件表达式支持：

```javascript
// 访问变量
{{score}} > 60

// 字符串匹配
{{name}} === "张三"

// 逻辑运算
{{age}} >= 18 && {{country}} === "中国"
```

### 2.2 文本分块原理

为什么要分块？

> 想象一下，你有一本1000页的书，想找到关于"如何养猫"的内容。如果整本书作为一个大段落搜索，可能会找到不精确的结果。但如果把书分成每页一段，就能精准定位到第234页提到了"养猫技巧"。

**分块参数**：

```typescript
{
  "chunkSize": 500,      // 每块最多500字符
  "chunkOverlap": 50,    // 相邻块重叠50字符
  "separator": ["\n\n", "\n", " ", ""]  // 分割符优先级
}
```

**分块示例**：

```
原文：
"人工智能是一门研究如何使计算机模拟人类智能的学科。它包括机器
学习、自然语言处理、计算机视觉等多个领域。近年来，随着深度学
习的发展，AI技术取得了突破性进展。"

分块结果：
┌──────────────────────┐  ┌──────────────────────┐
│ 块1: 0-500字符        │  │ 块2: 450-900字符      │
│ "人工智能是一门..."   │  │ "多个领域。近年来..." │
│ 索引: 0              │  │ 索引: 1              │
│ 重叠区: 后50字符      │  │ 重叠区: 前50字符      │
└──────────────────────┘  └──────────────────────┘
              ↓ 重叠50字符 ↓
```

### 2.3 向量检索原理

**什么是向量？**

把文字变成数字坐标，用高维空间表示语义：

```
二维简化示例：

     动物 ↑
          │
    猫 ●  │  ● 狗
          │
          │
──────────┼──────────► 宠物相关
          │
   汽车 ● │  ● 飞机
          │
          │
     交通工具

"猫"和"狗"距离近（都是宠物）
"猫"和"汽车"距离远（语义无关）
```

**实际维度**：OpenAI 的 embedding-3 模型输出 1536 维向量

**相似度计算**：

```sql
-- pgvector 的 <=> 操作符计算余弦距离
SELECT
  content,
  1 - (embedding <=> query_embedding::vector) AS similarity
FROM document_chunks
ORDER BY embedding <=> query_embedding::vector
LIMIT 10;
```

### 2.4 混合检索融合

为什么要混合？

| 检索方式   | 优点       | 缺点           |
| ---------- | ---------- | -------------- |
| 向量检索   | 找语义相关 | 可能漏掉关键词 |
| 关键词检索 | 精准匹配   | 不理解同义词   |

**融合公式（RRF）**：

```
score = (1 / (k + rank_vector)) + (1 / (k + rank_keyword))

其中 k=60 是平滑因子
```

**实际例子**：

```
问题："如何训练神经网络"

向量检索结果：
1. "深度学习中的神经网络训练方法" (rank=1)
2. "反向传播算法详解" (rank=2)

关键词检索结果：
1. "神经网络训练指南" (rank=1)
2. "深度学习中的神经网络训练方法" (rank=3)

融合后排序：
1. "深度学习中的神经网络训练方法" - 两种方式都找到
2. "神经网络训练指南" - 关键词找到
3. "反向传播算法详解" - 向量找到
```

### 3.2 SSE 流式输出原理

**为什么要流式？**

想象一下两种情况：

| 方式   | 体验                   | 举例     |
| ------ | ---------------------- | -------- |
| 非流式 | 等10秒后一次性显示全部 | 传统网页 |
| 流式   | 立即开始显示，逐字出现 | ChatGPT  |

**SSE（Server-Sent Events）**：

```
HTTP/1.1 200 OK
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive

data: {"token": "你好"}

data: {"token": "，"}

data: {"token": "我是"}

data: {"token": "AI"}

data: {"done": true}
```

**前端接收**：

```javascript
const eventSource = new EventSource('/api/chat/stream');

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.token) {
    messageContent += data.token; // 逐字追加
  }
};
```

### 3.3 引用来源展示

当 AI 回答基于知识库时，需要展示引用：

```
用户：公司的年假政策是什么？

AI：根据《员工手册》第三章规定，员工入职满一年后
可享受带薪年假5天。[1]

─────────────────────────
📚 参考来源：
[1] 《员工手册》第三章 - 第12页
    "带薪年假：员工累计工作满1年不满10年的，年休假5天..."
```

**实现方式**：

```typescript
// 保存消息时记录来源
{
  role: 'assistant',
  content: '根据《员工手册》...',
  sources: [
    {
      documentId: 'doc-123',
      documentName: '员工手册.pdf',
      chunkIndex: 5,
      content: '带薪年假：员工累计工作...',
      similarity: 0.92
    }
  ]
}
```

```
│                       用户操作                                 │

                    │
     ┌──────────────┼──────────────┐
     ▼              ▼              ▼

│设计工作流 │   │上传文档  │   │开始对话  │

     │             │              │
     ▼             ▼              ▼

│              后端处理                     │
│  ┌─────────────┐ ┌──────────┐ ┌────────┐ │
│  │保存节点/连线  │ │解析/分块  │ │RAG检索 │ │

                    │

     ▼              ▼              ▼

```
